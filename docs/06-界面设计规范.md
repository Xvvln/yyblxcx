# 六、界面设计规范

## 6.1 设计风格

### 参考对象

**主要参考**：

| 参考源 | 用途 | 位置 |
|-------|------|-----|
| **Keep App** | 运动健康类交互和布局 | `/design-references/` |
| **Apple App Store** | 前端设计风格、组件样式 | `/apps.apple.com-master/` |

**Keep App 参考要点**：
- 运动打卡流程
- 健康数据展示
- 社区动态布局
- 个人中心结构
- **Onboarding 流程（新）**：分步式问卷、沉浸式交互、自定义滑尺选择器

**App Store 参考要点**：
- 卡片圆角风格（23%/16px）
- 渐变遮罩效果
- 响应式布局
- 苹果设计系统的色彩和字体

### 设计原则

| 原则 | 说明 |
|-----|------|
| **简洁直观** | 核心功能 3 步内可达，参考 Keep 的简洁风格 |
| **运动感** | 体现健康、活力的视觉感受 |
| **适老化支持** | 提供「长辈模式」，默认关闭，用户可手动开启 |
| **情感化设计** | 打卡成功有动画反馈，连续打卡有成就感 |

## 6.2 强制规范

### 6.2.1 禁止使用 Emoji

**小程序内部一律禁止使用任何 Emoji 表情符号**，包括但不限于按钮、标题、提示信息等所有文案。

### 6.2.2 图标与图片占位

所有图标、图片位置**暂时使用占位符**，等待设计稿完成后统一替换：

```vue
<!-- 图标占位 -->
<view class="icon-placeholder" style="width: 48rpx; height: 48rpx; background: #E5E5E5; border-radius: 8rpx;"></view>

<!-- 图片占位 -->
<view class="image-placeholder" style="width: 100%; height: 200rpx; background: #F5F5F7;"></view>
```

- 占位图统一放置在 `/static/placeholder/` 目录

> 详细规范参见：[00-开发约束与说明.md](./00-开发约束与说明.md)

## 6.3 色彩指南

> **说明**：以下为推荐色彩，开发时可根据实际页面需求灵活调整，不做硬性规定。

### 主色调

| 名称 | 色值 | 用途建议 |
|-----|------|---------|
| 品牌蓝 | `#0071e3` | 主按钮、重要链接、品牌强调 |
| 活力绿 | `#34C759` | 运动相关、成功状态、健康指标 |

### 中性色

| 名称 | 色值 | 用途建议 |
|-----|------|---------|
| 纯白 | `#FFFFFF` | 页面背景、卡片背景 |
| 浅灰 | `#F5F5F7` | 次级背景、分隔区域 |
| 边框灰 | `#E5E5E5` | 分割线、输入框边框 |
| 深灰文字 | `#1D1D1F` | 标题、主要文字 |
| 浅灰文字 | `#86868B` | 辅助文字、说明文字 |

### 功能色

| 名称 | 色值 | 用途建议 |
|-----|------|---------|
| 成功 | `#34C759` | 成功提示、完成状态 |
| 警告 | `#FF9500` | 警告提示、注意事项 |
| 错误 | `#FF3B30` | 错误提示、删除操作 |
| 信息 | `#007AFF` | 信息提示、普通链接 |

### CSS 变量（建议）

```scss
// uni.scss 中定义
:root {
  // 主色
  --color-primary: #0071e3;
  --color-success: #34C759;
  --color-warning: #FF9500;
  --color-error: #FF3B30;
  
  // 背景
  --bg-page: #F5F5F7;
  --bg-card: #FFFFFF;
  
  // 文字
  --text-primary: #1D1D1F;
  --text-secondary: #86868B;
  --text-placeholder: #C7C7CC;
  
  // 边框
  --border-color: #E5E5E5;
}
```

## 6.4 页面功能说明

> **注意**：以下仅说明各页面需要包含的功能点，具体布局和排版参考 Keep App 截图。

### 首页
- 健康数据/评分展示
- 今日任务/打卡入口
- 快捷功能入口
- 推荐内容区域

### 运动页面
- 运动类型选择
- 实时数据显示（时长、配速、距离等）
- GPS 地图轨迹（参考 Keep 运动界面）
- 运动结束分享

### 饮食记录
- 餐次选择（早/中/晚/加餐）
- 食物拍照/识别
- 营养数据展示
- 饮食建议

### 健康筛查 (Onboarding)
- **分步式问卷**：每一步一屏，沉浸式体验
- **性别选择**：大卡片交互
- **身体数据**：使用自定义滑尺（RulerSlider）选择身高/体重
- **进度反馈**：顶部进度条 + 步骤指示
- **生成动画**：提交后展示分析进度动画

### 社区
- 动态流（参考 Keep/小红书风格）
- 关注/推荐切换
- 发布入口
- 互动功能（点赞、评论、分享）

### 商城
- 商品分类
- 商品列表/详情
- 购物车
- 订单流程

### 个人中心
- 用户信息卡片
- 功能菜单列表
- 数据统计入口
- 设置入口

## 6.5 交互建议

| 场景 | 交互方式 |
|-----|---------|
| 按钮点击 | 轻微缩放 + 颜色变化 |
| 页面切换 | 平滑过渡动画 |
| 列表加载 | 骨架屏或加载动画 |
| 下拉刷新 | 自定义刷新动画 |
| 打卡成功 | 庆祝动画 + 数据飞入效果 |
| 表单提交 | 按钮 loading 状态 |
| **数值选择** | 使用滑尺/滚轮代替简单输入框，提升手感 |

## 6.6 自定义组件规范

由于 Wot Design Uni 组件库可能无法完全满足 Keep 风格的交互需求，项目中引入了以下自定义组件：

### RulerSlider (刻度尺滑块)
- **用途**：用于身高、体重等连续数值的精确选择。
- **特性**：支持惯性滚动、中心吸附、实时数值反馈。
- **路径**：`components/ruler-slider.vue`

### GenderSelector (性别选择器)
- **用途**：用于性别选择。
- **特性**：大卡片样式，点击即选中，视觉反馈明显。
- **路径**：`components/gender-selector.vue`
