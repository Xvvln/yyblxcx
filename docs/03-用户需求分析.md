# 三、用户需求分析

## 3.1 用户角色定义

| 角色 | 描述 | 权限 |
|-----|------|-----|
| **普通用户** | 使用小程序进行健康筛查、记录饮食运动、购买商品、参与社区互动 | 上传数据、查看报告、接收建议、购物、打卡、评论、点赞、添加好友 |
| **会员用户** | 购买会员服务的用户，享受更多功能与权益 | 解锁高级报告、专属膳食/运动计划、积分翻倍、优先客服 |
| **医生/营养师** | 平台合作的专业人员 | 查看用户匿名数据、回复咨询、发布科普内容、标记高风险用户 |
| **平台管理员** | 负责系统运营、内容审核 | 管理用户、审核内容、配置积分规则、发布活动、查看数据报表 |

## 3.2 功能需求清单

### 3.2.1 核心功能

| 编号 | 功能描述 |
|-----|---------|
| F01 | 营养不良风险筛查（问卷+图像识别+膳食记录） |
| F02 | 基于筛查结果自动生成个性化健康管理建议（膳食+运动） |
| F03 | 每日饮食与运动数据记录 |
| F04 | 健康商城（商品浏览、下单、支付） |
| F05 | 会员系统（月/年/终身卡） |
| F06 | 运动课程选择、食物课程选择 |

### 3.2.2 增强体验功能

| 编号 | 功能描述 |
|-----|---------|
| F07 | 社区互动（发布动态、点赞评论、关注好友、运动打卡分享） |
| F08 | 积分激励（运动币/膳食币，可兑换商品或优惠券） |
| F09 | 多种运动模式（跑步、骑行、跳绳、健走，支持GPS轨迹记录与配速分析） |
| F10 | 智能提醒（喝水、运动、吃药、吃饭提醒） |
| F11 | 远程医疗入口（对接合作医院，图文/视频咨询） |

### 3.2.3 扩展功能

| 编号 | 功能描述 |
|-----|---------|
| F12 | 家庭账号管理（一户多号） |
| F13 | AI健康助手（自然语言交互） |
| F14 | 第三方设备接入（智能手环、体重秤、血糖仪） |
| F15 | 健康挑战赛（21天打卡挑战等） |
